# Code Style Rules

## Backend (PHP / Symfony)
- PSR-12 compliance
- Strict types required
- No business logic in controllers
- Services must be injectable
- Use DTOs for all input
- Restful API conventions

## Frontend (React / TS)
- No `any`
- Functional components only
- Tailwind only (no inline styles)
- react-hook-form mandatory for forms
- React 19 approach
- SOLID principle of working/development (Most Single Responsibility)

## General
- No magic strings
- Use enums/constants
- No dead/commented code (only when necessary)
